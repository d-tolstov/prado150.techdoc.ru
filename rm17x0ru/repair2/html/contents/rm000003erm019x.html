КРАТКОЕ ОПИСАНИЕ

  1. CAN (Controller Area Network – сеть контроллеров) – это последовательная система обмена данными, работающая в реальном времени. Система мультиплексной связи автомобиля имеет высокую скорость передачи данных (500 кбит/с) и функции обнаружения неисправностей.

  2. В системе CAN данные передаются дифференциальным напряжением между линиями шины CANH и CANL.

  3. Большинство установленных на автомобиле блоков ЭБУ (датчиков) работают, поддерживая обмен данными друг с другом.

  4. Шина CAN имеет 2 резистора сопротивлением 120 Ом, которые необходимы для обмена данными с использованием главной шины.


ОПРЕДЕЛЕНИЕ ТЕРМИНОВ

  1. Главная шина

    1. Главной шиной называется жгут проводов между двумя конечными цепями на шине (линия связи). Это – главная шина в системе передачи данных CAN.

  2. Вспомогательная шина

    1. Вспомогательной шиной называется жгут проводов, отходящий от главной шины к ЭБУ или датчику.

  3. Конечные цепи

    1. Конечная цепь преобразует ток сигнала передачи данных CAN в напряжение шины. Она состоит из резистора и конденсатора. На шине должно быть две конечных цепи.

  4. Разъем распределительного блока шины CAN

    1. Распределительный блок шины CAN сконструирован так, что он может использоваться для передачи данных CAN, имеющей конечные цепи, и для передачи данных CAN, которая не имеет конечных цепей.


ОПИСАНИЕ ЦЕПИ

  1. Шина V1 имеет конечные цепи с сопротивлением 120 Ом х 2. Таким образом обеспечивается высокоскоростная передача данных со скоростью 500 кб/с.


ЭБУ ИЛИ ДАТЧИКИ, КОТОРЫЕ ОБМЕНИВАЮТСЯ ДАННЫМИ ПОСРЕДСТВОМ СИСТЕМЫ ПЕРЕДАЧИ ДАННЫХ CAN

  1. ECM*1

  2. Главный ЭБУ кузова (бортовой ЭБУ сети мультиплексной связи)

  3. Блок управления рабочими цилиндрами тормозов в сборе (ЭБУ системы противоскольжения)

  4. ЭБУ полного привода

  5. Датчик рысканья в сборе

  6. DLC3

  7. Витой кабель в сборе (датчик угла поворота рулевого колеса)*2

  8. ЭБУ рулевого управления с усилителем*1

  9. Дисплей модуля системы навигации и индикации (AVN)*3

  10. Центральный блок управления системы SRS

  11. Щиток приборов в сборе

  12. Блок управления системой кондиционирования в сборе

    1. *1: для моделей с 2TR-FE
    2. *2: для моделей с системой курсовой устойчивости (с вакуумным усилителем тормозной системы)
    3. *3: Для моделей с системой навигации


ДИАГНОСТИЧЕСКИЕ КОДЫ ДЛЯ СИСТЕМЫ ПЕРЕДАЧИ ДАННЫХ CAN

  1. Коды DTC для системы передачи данных CAN: U0073, U0100, U0114, U0123, U0124, U0126, U0129, U0131, U0142, U0151, U0155, U0164


ПРИМЕЧАНИЯ ПО ДИАГНОСТИКЕ

  1. Неисправность на шине CAN (линии связи) можно определить посредством DLC3 (за исключением тех случаев, когда имеется обрыв на вспомогательной шине DLC3).

  2. Коды DTC для системы передачи данных CAN можно проверить, используя портативный диагностический прибор.

  3. Система передачи данных CAN не может обнаруживать неисправности вспомогательной шины DLC3, несмотря на то, что DLC3 также подключен к системе CAN.


КАК РАЗЛИЧАТЬ РАЗЪЕМЫ РАСПРЕДЕЛИТЕЛЬНОГО БЛОКА ШИНЫ CAN

  1. В системе передачи данных CAN все разъемы, подключенные к распределительному блоку шины CAN, имеют одинаковую форму.
    Разъемы, подключенные к распределительному блоку шины CAN, можно различать по цвету шин и соединительной части разъема.
